# 01_cover
这份招商方案基于岚禾小食现有静态商城和新增管理后台计划整理。核心目标不是单纯展示几款宠物零食，而是把天然产品、购买路径和经营数据组织成一套合作方能理解、能演示、能复盘的招商材料。

# 02_investment_thesis
岚禾小食的招商价值来自三个“快”：合作方能快速讲清产品，门店能快速完成分类陈列，招商现场能快速演示从购买到后台统计的闭环。这里没有虚构市场规模，而是把网页和代码里已经具备的功能转译为渠道合作价值。

# 03_product_standards
天然概念必须变成可复述的话术。产品详情页里的成分、适用对象、适口性、标签和卖点，是导购和私域内容最容易复用的素材。对渠道而言，这降低了培训成本，也减少了只靠包装图卖货的不确定性。

# 04_product_matrix
四类主推产品对应四个常见购买场景。鸡肉条适合日常奖励，冻干适合搭配主粮，洁齿棒适合功能型复购，训练粒适合互动和社群活动。这样的货盘结构便于小规模试销，也便于后续按销量加深 SKU。

# 05_scene_logic
宠物零食的销售不是一次性的“买一包”，而是围绕宠物家庭的日常关系展开。奖励、加餐、护理和训练分别对应不同购买理由。招商沟通时，先讲场景，再讲产品，会比直接堆 SKU 更容易让合作方判断适配度。

# 06_storefront_flow
前台商城已经可以演示完整购买链路：分类筛选、查看详情、加入购物车、填写地址、生成订单。关键是，下单后会写入本地订单，后台能同步看到这条记录。这让招商演示从静态页面变成可操作流程。

# 07_admin_loop
后台的价值在于把产品展示推进到经营展示。订单列表、营业额、订单数、售出件数和销量排行，可以让合作方看到“卖什么、卖了多少、接下来补什么”。新品上架也能同步出现在前台，形成一套演示沙盘。

# 08_channel_fit
不同渠道关注点不同。宠物门店看重货架动销，区域代理看重招商说服力，私域团长看重内容转化，品牌合作方看重低成本验证。岚禾小食可以用同一套商品和演示系统，对应不同合作入口。

# 09_launch_steps
合作启动可以做轻：先选 2 到 4 个主推 SKU，再按场景做陈列和话术，最后用订单和销量数据做复盘。这个节奏适合试销，不需要一开始就承诺复杂系统或大库存。

# 10_closing
最后的合作主张是：把天然宠物零食做成可复购的渠道生意。岚禾小食提供的不只是产品图片和价格表，而是一套能被讲清、被演示、被复盘的合作基础。
