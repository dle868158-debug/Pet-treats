# Pet Treats

一个面向高端天然宠物零食品牌的静态展示与前端购物车示例站点。页面以中文内容呈现，包含入口登录、游客浏览、商品筛选、热销榜、商品详情弹窗、购物车抽屉、账户中心、多地址管理和本地模拟下单流程。

## 功能特性

- 品牌首页首屏展示
- 两级商品筛选：先选全部宠物、猫猫、狗狗，再选食品分类
- 食品分类动态展示：只显示当前宠物分类下可用的鸡肉、冻干、洁齿、训练奖励等分类
- 热销榜：根据商品销量自动排序展示前三名
- 商品卡片：展示标签、价格、销量和简介
- 商品详情弹窗：展示成分、适用建议和适口性
- 购物车功能：加入商品、增减数量、删除商品、合计金额
- 本地购物车持久化：使用 `localStorage` 保存已选商品
- 登录注册入口：进入站点前先选择登录、注册或游客浏览
- 游客模式：可以浏览和加购，点击购买时会直接跳转登录页
- 账户中心：登录后管理多个收货地址、设置默认地址、删除地址
- 购买前确认地址：点击“立即购买”后弹出确认收货地址窗口，可选择默认地址、新增地址或设置默认地址
- 模拟提交订单：前端生成模拟订单号
- 管理后台：查看订单列表、营业额、商品销量排行和销售明细
- 新品上架：后台新增商品后保存到本地，并可在前台刷新后参与筛选、详情和购物车流程

## 本地运行

这是一个纯静态项目，不需要安装依赖。

直接用浏览器打开入口页：

```text
auth.html
```

也可以在项目目录下启动任意静态服务器，例如：

```bash
python -m http.server 8080
```

然后访问：

```text
http://localhost:8080
```

## 账号与地址说明

当前版本没有后端服务，账号、登录状态、游客状态、购物车和收货地址都保存在浏览器 `localStorage` 中，仅用于原型演示：

- 首次访问首页会先跳转到登录入口页
- 用户可以注册 / 登录，也可以选择游客身份进入站点
- 游客可以浏览商品和加入购物车，但点击“立即购买”时会跳转回登录页
- 注册账号后会自动登录
- 从购物车结算入口跳转登录后，会自动返回首页并打开购物车
- 登录后点击“账户中心”可以新增多个收货地址
- 地址可以设置默认、删除；购买前会在确认地址弹窗中选择本次收货地址
- 确认购买后，订单会写入 `localStorage["pawNaturalOrders"]`
- 后台新增商品会写入 `localStorage["pawNaturalProducts"]`

真实上线时应将账号认证、密码存储、地址管理和订单提交迁移到后端服务，不能把密码明文保存在浏览器本地。

## 商品数据维护

商品数据集中维护在 `script.js` 的 `products` 数组中。新增或调整商品时，主要修改以下字段：

```js
{
  id: "chicken-strips",
  name: "慢烘鸡胸肉条",
  category: "鸡肉",
  petTypes: ["cat", "dog"],
  price: 59,
  sales: 2680,
  weight: "120g",
  image: "assets/product-chicken-strips.png",
  tags: ["高蛋白", "低脂", "猫狗通用"],
  ingredients: "...",
  suitability: "...",
  palatability: "...",
  description: "..."
}
```

热销榜会根据 `sales` 字段自动排序，无需单独维护榜单内容。

`petTypes` 用于控制宠物分类筛选和商品适用标识：

- `["cat", "dog"]`：猫狗通用
- `["cat"]`：猫猫适用
- `["dog"]`：狗狗适用
